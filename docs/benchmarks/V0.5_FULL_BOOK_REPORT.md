# v0.5.1 full-book local benchmark

Run date: 2026-07-19

This benchmark exercises the current generic local pipeline on a 125-page Japanese manga at its native 2126 × 3661 pixel geometry. The run used no online LLM API and no book-specific or manually maintained person-name table.

## Runtime configuration

- OCR geometry: PaddleOCR with Manga OCR recognition
- Semantic OCR and context review: `huihui_ai/qwen3.5-abliterated:9b` through local Ollama
- Independent translation candidate: Hy-MT2 1.8B
- Name consistency: recurring-name extraction plus JMnedict lookup and local 9B disambiguation
- Artwork repair: Big-LaMa with type-size-aware outlined-text halos and bounded edit regions
- Output: native-resolution lossless WebP
- Sound-effect policy: preserve source SFX

The three README comparisons are generated from pages 1–3 of this output with `scripts/build_readme_comparisons.py`. The comparison files are resized for GitHub only; the benchmark output itself remains 2126 × 3661.

## Verification command

```powershell
uv run python scripts/verify_output.py SOURCE OUTPUT TRANSCRIPT --preserve-sfx --output-format webp
```

The verifier rejects missing pages, zero-byte images, dimension changes, unresolved or kana-bearing Chinese fields, mismatched output formats, and any decoded pixel change outside the declared per-unit edit masks.

## Scope

This is a regression result, not a claim that OCR or literary translation can be perfect on every drawing style. The pipeline intentionally preserves uncertain stylized sound effects instead of erasing artwork on unconfirmed VLM geometry. All detection, filtering, translation, and rendering rules are based on page and text-unit features; none contain this book's title, dialogue, page numbers, coordinates, or a fixed cast list.
